## Context Handoff

### Session Intent

Create a stable Forge testing strategy that separates deterministic workflow e2e from optional live task-board smoke validation.

### Key Files

- `package.json`
- `README.md`
- `tests/forge-runner.test.mjs`
- `tests/mcp-server.test.mjs`
- `tests/task-source-mcp.test.mjs`
- `tests/task-source-gitlab.test.mjs`
- `tests/task-label-sync.test.mjs`
- `tests/forge-board-labels.test.mjs`
- Optional: `tests/task-source-live-smoke.test.mjs`

### Decisions Made

- Default regression stays deterministic and uses mocked task-board behavior.
- Live tracker validation is opt-in smoke coverage, not required for every e2e run.
- Windows users should run scripts through `npm.cmd` when PowerShell blocks `npm.ps1`.

### Risks

- Script names can drift from README if not verified together.
- A live-board smoke test can become flaky if it validates too much external state.
- Existing draft plans around GitLab fail-fast and task gating overlap in behavior; this track should depend on them, not reopen their implementation scope.
